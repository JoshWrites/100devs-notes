“Brain Freezing Up When Encountering New Tools and Unexpected Bugs. Resources to Recommend?”
Brain Freezing up is likely a mindset shift
Biggest mindset shift for Leon, especially for non-deterministic stuff
Used to ask when he hit a bug: “What went wrong?”
Now thinks this is a mistake
Now: “I wrote the code in a particular way. Why was this the outcome?”
Slight difference, but now the bug is the by-product of what he has done. It’s not something that randomly happened to him. It’s something he did that led to a result
Did he do something differently? Is the bug repeatable? Does he keep getting the same error?
Realizing that he generated the bug helps empower him to know that he can also find a way to fix it and, ideally, not generate this type of bug again
Getting Good at Debugging as a Skill
Good to have process you go through. Pre-flight checklist, like if you were flying a plane
Should have something similar when debugging. Checklist of steps you walk through each time
Get used to using debugging tools. Use a debugger, logger, print statements, etc. to your advantage to get more clarity around what is happening
Get better at reading through the flow of information and all the touchpoints of your code to see how the error is being produced
Mindset that it’s fixable, checklist that you work through, and use of the appropriate tools
You now have a system to work through that can make the process of debugging much easier.
Typically 90% of what is going to happen to you is one of ten or so different things
Async thing? State thing? Only so many things that can go wrong.
If you’re able to repeat it, you can learn how to narrow down which thing is likely happening
GPT can give you a road map about debugging methods and what to look for. It’s definitely a learnable skill, and so we can find debugging techniques that have worked for people that are likely to work for us, too
Much of debugging is about practice and experience. The more we do it and stick to an effective checklist, the more the issues surface in a familiar way, we get more practice at getting familiar with a process that works, so each time we debug it can be a little easier and we can be more confident in our capacity
